CRC Test Data¶

Boost Libraries¶

Boost is not part of the Yukon firmware nor the SDK. It is being used to generate Yukon firmware unit tests. To compile the test data module requires boost to be in the compiler include path. This test data is compiled using boot version 1.79.0, which is installed in the directory /opt/boost_1_79_0.

Therefore, the gcc command line required to build the test data requires the -I /opt/boos_1_79_0 option.

CRC-5/EPC¶

The CRC-5/EPC, also named CRC-5/EPC-C1G2, is the CRC-5 encoder/decoder specified in Annex F of the Gen2 specification.

References:

  1. https://reveng.sourceforge.io/crc-catalogue/all.htm#crc.cat-bits.5

  2. https://en.wikipedia.org/wiki/Cyclic_redundancy_check#CRC_catalogues

  3. https://pythonhosted.org/crccheck/crctable.html

The CRC-5/EPC is succinctly specified as:

  • name = “CRC-5/EPC-C1G2”

  • width = 5

  • poly = 0x09

  • init = 0x09

  • refin = false

  • refout = false

  • xorout = 0x00

  • check = 0x00

  • residue = 0x00

(See reference 1 above)

This specification can be used with the boost library to instantiate CRC instances as:

boost::crc_optimal<5, 0x09, 0x09, 0x00, false, false> crc_5_epc_fast;
boost::crc_basic<5> crc_5_epc_basic(0x09, 0x09, 0, false, false);

These are used to generate the CRC test data used to verify the CRC-5/EPC implementation used in the Yukon firmware.
