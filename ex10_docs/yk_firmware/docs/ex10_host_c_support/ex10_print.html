Ex10Print¶

The Ex10Print layer is a wrapper function around a normal printf(). These wrapper functions can be enabled or disabled at build time with the following defines, that have default definitions in the top level ex10_c_dev_kitCMakeList.txt file, and can be overridden on the build command line with -D EX10_ENABLE_PRINT=OFF for example.

  • EX10_ENABLE_PRINT

  • EX10_ENABLE_PRINT_ERR

  • EX10_ENABLE_PRINT_EX

  • EX10_ENABLE_PRINT_EX_ERR

This allows these printf() commands to be enabled/disabled at compile time as desired. And the implementation of the ex10_eprintf_impl() call allows for the printf() functionality to be redirected in systems that do not have a normal stdout, like a microcontroller that has a UART output for debug for example.
