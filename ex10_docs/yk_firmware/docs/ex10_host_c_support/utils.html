Ex10 Utils¶

The Ex10 Utils are a collection of common code functions that are used by different parts of the SDK and application. These functions are intended to be global functions with an ex10_ prefix so that they can be included in an application individually. As opposed to the other modules that use a struct to collect up a group of function pointers. This forces the linker to include all of those functions in the application event if they are not not actually used. These functions are stand alone functions that do a simple common task.

Defines

ZERO_ACCESS_PWD¶

The zero password is the default password of tags. It is also the value to restore the tag to in examples.

NON_ZERO_ACCESS_PWD¶

The non_zero password is used for all examples to align to one non-zero password.

ssize_t ex10_discard_packets(bool print_packets, bool flush_packets, bool debug_aggregate_op)¶

Purge FIFO of remaining packets.

Print summary of each packet

Parameters

  • print_packets – Print each packet purged from the EventFifo queue.

  • flush_packets – Uses the InsertFifoEvent command, with the Trigger Interrupt flag set, to trigger an ReadFifo command, allowing the host to receive all pending EventFifo data held by the Reader Chip.

  • debug_aggregate_op – Provides detailed information about an AggregateOp failure, if an AggregateOpSummary packet is encountered in the dumped packet stream.

Returns -1

Indicates ~~an error occurred when emptying the FIFO.

Returns

ssize_t On success, the number of EventFifo packets discarded.

struct Ex10Result ex10_copy_tag_read_data(struct TagReadData *dst, struct TagReadFields const *src)¶

Copy the EventFifo TagRead extracted data into the destination type struct TagReadData.

The struct TagReadFields contains data pointers into the relevant fields of the EventFifo TagRead packet, allowing for easy data extraction. The EventFifo packet will be deleted when the function Ex10Reader.packet_remove() function is called, which releases the EventFifo memory back to the pool for reuse. The struct TagRead provides a destination for the TagRead data, allowing the client to use the tag data beyond the lifetime of the EventFifo TagRead packet.

Returns true

indicates that the tag was copied without error.

Returns struct

Ex10Result fails if either the EPC or the TID length was larger than the struct TagReadData allocation. To determine which copy failed, the destination length fields should be compared with the source length fields.

Returns

bool Indicates whether the copy operation was successful.
