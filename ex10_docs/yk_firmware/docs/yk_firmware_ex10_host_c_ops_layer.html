Ex10Ops Layer¶

The Ex10Ops layer provides an operation-level interface that allows the user to start atomic operations, known as ‘Ops’, on the device. Atomic operations are fundamental actions that can be executed by the Impinj reader chip. Some atomic operations produce data in the EventFifo. As an example, the StartInventoryRoundOp is an op started through the start_inventory_round(...) function and will produce an EventFifo packet InventoryRoundSummary. This function take arguments which are used to set up the Ex10 registers before running the op.

SDK Aggregate Operations are an aggregation of atomic operations as well as register writes used to set up the op before running. The Ops layer does not wait for these atomic operations to complete before returning to allow the user time to perform other actions. To ensure an op is complete, higher layers should check the OpStatus register or utilize the Ops layer function wait_op_completion. The Impinj Reader Chip Host Library supports the following operations:

struct Ex10Result (*set_aux_dac)(uint8_t dac_channel_start, uint8_t num_channels, uint16_t const *dac_values)¶

This executes the SetDacOp operation which writes values to the AUX DAC for each channel specified.

When the SetDacOp has completed the digital to analog conversions will have completed.

Note

There are 2 AUX DAC channels which can be written.

  • If a channel is omitted from the DAC list then that channel is disabled and its output will be zero.

  • If more channels are set outside the range of the DAC list, then the out of bounds values are ignored.

Param dac_channel_start

Determines the starting DAC channel.

Param num_channels

The number of DAC channel conversions to perform.

Param dac_values

An array of DAC channel data containing unsigned 10-bit values for conversion. The size of this array must be uint16_t[num_channels].

Return

struct Ex10Result Indicates whether the function call passed or failed.

struct Ex10Result (*run_listen_before_talk)(struct LbtControlFields const *lbt_settings, struct RxGainControlFields const *used_rx_gains, struct LbtOffsetFields const *lbt_offsets, struct RfSynthesizerControlFields const *rf_synth_control, uint8_t rssi_count)¶

Runs the listen before talk op.

The op goes through the following steps:

  1. Sets up the modem and internal low pass filter

  2. Set the RF synthesizer using the RF Synth control and offset frequency

  3. Run the RSSI measurement.

  4. Delay for the user specified time.

  5. Repeat 2-4 the number of user specified times using an index incrementing from 0.

Note

: Before running LBT, the host must have run 2 ops: set_gpio, set_analog_rx_config.

Note

: CW must be ramped down before LBT

Note

: The op leaves the synthesizer locked to the last frequency used when done.

Param lbt_settings

Controls for the LBT op including the number of rounds to run and the delay between each round.

Param lbt_offsets

The offset frequency to use for each measurement. Note that this is an array which can differ for each measurement. Ensure that each index is set up to the number of measurements needed. If taking 3 measurements, set index 0-2.

Param rf_synth_control

The PLL controls to use for each measurement. Note that this is an array which can differ for each measurement. Ensure that each index is set up to the number of measurements needed. If taking 3 measurements, set index 0-2.

Param rssi_count

The integration count to used for the RSSI block

Return

struct Ex10Result Indicates whether the function call passed or failed.
