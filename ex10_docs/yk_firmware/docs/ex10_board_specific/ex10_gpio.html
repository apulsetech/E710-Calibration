GPIO Abstractions¶

In a reader implementation there are typically other devices that control different aspects of the readers operation. This can include things like antenna muxes, TX or RX filters, or board power controls. These GPIOs can be controlled by the host processor or the GPIOs provided by the Impinj reader chip. In the Impinj E710 development kit there are several of these controls and the GPIO interface features below help to abstract them so that another reader design can use the same API but have different functionality associated with the features.

The Ex10GpioInterface structure below is a top level structure that represents the GPIO interface on the host that is used to control and communicate with the Impinj reader chip. The board implementation must implement these functions so that the SDK can communicate with the Impinj reader chip. The power and enable functions must be implemented, but can be stubs if the reader is designed to have the Impinj reader chip powered up and enabled by default.

The Ex10GpioDriver structure is the driver for the host GPIO pins.

The Ex10GpioConfig is a structure that is defined by the reader for the GPIO pins that are controlled by the Impinj reader chip. This structure is then used by the SDK to set the Impinj reader chip GPIOs for the required condition. The Ex10GpioHelpers structure contains typical helper functions that are used to manipulate the Impinj reader chip GPIOs for setting different conditions in the reader. They are up to the board layer to implement as needed for the hardware. These functions may need to do something and not change the GPIO settings at all, such as controlling an antenna mux by a host SPI interface instead of GPIOs on the Impinj reader chip. In this situation it would do the required SPI transaction, and then not modify the gpio_pins_set_clear structure that the SDK will use to set the Impinj reader chip GPIOs.

int32_t (*gpio_initialize)(bool board_power_on, bool ex10_enable, bool reset)¶

Acquire ownership of the Ex10 connection pins from the the GPIO driver and initialize their state.

i.e Set the pins’ directions and levels.

Note

If this function returns with a failure, resources will likely have been allocated, and not released. The caller should free resources by calling gpio_cleanup() to free these resources.

Param board_power_on

If true, asserts the PWR_EN line to enable the the Impinj Reader Chip power supply. If false, power is disabled to the Reader Chip.

Param ex10_enable

If true the Ex10 ENABLE is asserted high. If false the Ex10 ENABLE line is deasserted low.

Param reset

If true the RESET_N line is asserted low. if false the RESET_N line is deasserted high.

Return

int32_t Indicates success or failure. Zero for success, non-zero for failure.

void (*gpio_cleanup)(void)¶

Release the GPIO driver and its resources from usage.

In the reference design, this will:

  • Destroy and join the IRQ_N monitor thread.

  • Release and close resources requested from the GPIO library.

int32_t (*set_ex10_enable)(bool enable)¶

Assert the ENABLE line high or deassert the ENABLE line low.

When the ENABLE line is low the EX10 device will internally power down. When the ENABLE line transitions from low to high the Ex10 device executes a POR (power-on-reset).

Note

If the power_on parameter is true, this call will not return until the RESET_N pin is high (‘1’).

Param enable

When true the ENABLE line is assert high; when false the ENABLE line is deasserted low.

Return

int32_t Indicates success or failure. Zero for success, non-zero for failure.

bool (*get_ex10_enable)(void)¶

Get the ENABLE line state.

Return

bool Indicates the ENABLE line level. true: The line level is logical high (1). false: The line level is logical low (0).

int32_t (*register_irq_callback)(void (*cb_func)(void))¶

Registers a callback function which occurs on the falling edge of the IRQ_N_PIN and is processed within the irq_n_monitor thread context.

Successful registration of the interrupt callback will start the IRQ_N monitor thread, which will call back within this thread context to the client supplied cb_func parameter when IRQ_N asserts low.

Param cb_func

The function that is called when IRQ_N asserts low.

Return

int32_t Indicates success or failure. Zero for success, non-zero for failure.

int32_t (*deregister_irq_callback)(void)¶

Deregister the callback function from the IRQ_N monitor thread.

This will destroy the IRQ_N monitor thread and remove the callback entry from the gpio driver.

Return

int32_t Indicates success or failure. Zero for success, non-zero for failure.

void (*irq_monitor_callback_enable)(bool enable)¶

Enable or disable the IRQ_N monitor thread to call the registered callback function.

Param enable

If set to true then the registered callback function will be called, within the IRQ_N monitor thread context, when an IRQ_N falling edge occurs.

bool (*irq_monitor_callback_is_enabled)(void)¶

Determine whether the IRQ_N monitor thread will call the registered callback in response to interrupt activity.

Return

bool true: The registered callback will be called when the IRQ_N falling edge is detected. false: The registered callback will not be called when the IRQ_N falling edge is detected.

void (*irq_enable)(bool enable)¶

Locks or unlocks access to the host (SPI) and GPIO interfaces to guard against pre-emptive access to the hardware interfaces.

Note

In the reference design, a POSIX non-counting mutex is used to perform this lock.

Param enable

If set to false, the hardware interfaces are locked for use by the caller. If set to true, the hardware interfaces are released for use.

struct Ex10GpioConfig¶

E710 Evaluation board GPIO levels mapping: Antenna: 2: ‘0’, 1: ‘1’ Baseband Filter: Highpass: ‘0’, Bandpass: ‘1’ PA Bias Enable: Disable: ‘0’, Enable: ‘1’ PA Bias Range: Low: ‘0’, High: ‘1’ RF Enable: Disable: ‘0’, Enable: ‘1’ SAW Filter: 900-930 MHz: ‘0’, 865-868 MHz: ‘1’.

(forward declaration)

Each board level implementation will need to provide its own specific declaration of the Ex10GpioConfig depending on what functionality the board provides.

param gpio_config

Board agnostic settings which require GPIO levels to be set to specific levels.

struct Ex10GpioHelpers¶

Converts a structure of GPIO functional settings into struct GpioPinsSetClear which can be used to set the SetClearGpioPinsOp registers.

Each of these function calls combines the struct GpioPinsSetClear parameter, using bit-wise OR, with the gpio_pins_set_clear values passed in. In this manner successive calls can be combined into a single run of SetClearGpioPinsOp.

struct GpioPinsSetClear¶

Used to control output state of each DIGITAL_IO pin of the Impinj Read Chip.

Public Members

uint32_t output_level_set¶

Set a group of DIGITAL_IO pins to have their outputs set to Logic 1. This value will only be preset on pins which have their outputs enabled.

uint32_t output_level_clear¶

Set a group of DIGITAL_IO pins to have their outputs set to Logic 0. This value will only be preset on pins which have their outputs enabled.

uint32_t output_enable_set¶

Set a group of DIGITAL_IO pins to be enabled as output pins.

uint32_t output_enable_clear¶

Disable a group of DIGITAL_IO pins from being enabled as output pins. Pins which have their output disabled will have the Hi-Z state.
