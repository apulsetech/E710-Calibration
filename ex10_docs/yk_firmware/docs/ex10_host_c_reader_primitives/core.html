The Ex10 Core¶

The Ex10 core is the core group of modules that are needed for any communication with the Impinj reader chip for any purpose. These include modules such as Ex10Ops and Ex10Protocol and the additional modules that they build on. The ex10_core_board_setup() function is a helper to assemble and initialize these core modules. There is also a ex10_bootloader_core_board_setup() that is used to bring the Impinj reader chip up into the bootloader.

Functions

struct Ex10Result ex10_core_board_setup(enum Ex10RegionId region_id, uint32_t spi_clock_hz)¶

Initialize the Impinj Reader Chip core SDK.

This includes the Ops and protocol layers, and any board bring up.

Parameters

  • region_id – The region to initialize the active region with

  • spi_clock_hz – The SPI inteface clock speed in Hz

Returns

struct Ex10Result Indicates whether the function call passed or failed.

struct Ex10Result ex10_bootloader_core_board_setup(uint32_t spi_clock_hz)¶

Initialize the Impinj Reader Chip and its associated Ex10 host interfaces in a minimal configuration for communication with the bootloader.

This initialization profile is primarily used for uploading new firmware images into the Impinj Reader Chip.

Parameters

spi_clock_hz – The SPI interface clock speed in Hz. This value must be 1 MHz or less.

Returns

struct Ex10Result Indicates whether the function call passed or failed.

void ex10_bootloader_core_board_teardown(void)¶

Deinitailize the Ex10Protocol object and power down the Impinj Reader Chip when the Impinj Reader Chip was configured in bootloader execution status.

void ex10_core_board_teardown(void)¶

Deinitialize the Ex10 Core interface groups of objects and power down the Impinj Reader Chip.

This is strictly a software and driver layer release of host resources. No transactions are performed to the Impinj Reader Chip over the host interface within this function call.

void ex10_core_board_gpio_init(struct Ex10GpioInterface const *gpio_if)¶

Initialize the gpio_driver pins required for Impinj Reader Chip operation.

The following schematic pins are set as output direction, with the levels: PWR_EN deassert, active high, pin level = 0 ENABLE deassert, active high, pin level = 0 RESET_N assert, active low, pin level = 0

Parameters

gpio_if – The instance of the GpioInterface.

Variables

static uint32_t const DEFAULT_SPI_CLOCK_HZ = 4000000u¶

Typical usage is 4MHz when running the Application.

static uint32_t const BOOTLOADER_SPI_CLOCK_HZ = 1000000u¶

A reduced clock speed is required when running the Bootloader.
