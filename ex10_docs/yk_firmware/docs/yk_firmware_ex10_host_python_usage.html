Python SDK usage¶

The Python SDK makes use of the ctypes library to reference functions within the C SDK. Therefore, the C SDK lib_p2yc.so shared object must be built and installed prior to using the Python SDK.

Setting up the Python virtual environment¶

To set up the python virtual environment:

$ cd ex10_dev_kit
$ source source_me_for_impinj_reader_chip_venv.sh
** installing requirements **
Looking in indexes: https://pypi.org/simple, https://www.piwheels.org/simple
# ... Installation takes some time ...
(impinj_reader_chip_venv) $

This will install the Python interpreter along with all of the necessary packages required to run the Impinj Reader Chip Python SDK.

Once the python virtual environment is activated, the shell command prompt will be prefixed with (impinj_reader_chip_venv). This lets the use know that the virtual environment is activated.

To deactivate the python virtual environment:

(impinj_reader_chip_venv) $ deactivate
$

The prompt will no longer have the prefix and the Python virtual environment will no longer be in use.

After the virtual environment is installed, activating the virtual environment can be more quickly activated use the activate command:

$ cd ex10_dev_kit
$ source impinj_reader_chip_venv/bin/activate
(impinj_reader_chip_venv) $

This activates the Python virtual environment much more quickly. Activating the Python virtual environment using the installation command source source_me_for_impinj_reader_chip_venv.sh will also work; it just takes longer to complete.

Running Python SDK examples¶

Testing that the Python installation was successful can be done by running any of the Python SDK examples, i.e.:

(impinj_reader_chip_venv) $ python3 ./examples/inventory_example.py
