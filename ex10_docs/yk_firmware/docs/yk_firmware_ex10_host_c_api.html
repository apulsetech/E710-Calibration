Impinj Reader Chip C Host SDK¶

The host C library is made up of several C source files, implementing different aspects of the Impinj reader chip upper MAC. This host C library is used in the implementation of the provided C code examples.

Previous releases of the SDK provided a host API that consisted of a monolithic layer called the ‘Ex10Reader’ that, along with its helper functions, provided a plethora of examples. However these examples, and helpers, made a lot of assumptions about the HW platform, and thus were difficult to separate and build from, as well as resulting in unoptimized code size. While we still provide this approach for backward compatibility reasons, we recommend moving to our new, more customizable, and optimized, approach we call ‘Use Cases’.

The new ‘Use cases’ approach provides several common ‘use case’ examples, or behaviors, from which a developer can build their application around. ‘Use cases’ are more easily tailored to improve the developer experience, providing more control, and reduce code size.

The ‘Use Case’ layer, is a collection of common reader behavior patterns to begin building your application. It exposes an interface that implements highly integrated RAIN RFID functionality that are targeting specific scenarios, such as running inventory continuously, or singulating tags and interacting with the tag.

The host library also consists of a collection modules organized into the following groups:’Reader Primitives’, ‘Ex10 Core’ and ‘Ex10 Support’.

The ‘Reader Primitives’ is a collection of modules that implement specific RAIN RFID primitives such as: ramping up power, or running inventory rounds. Only the necessary primitives required to implement a ‘use case’ are required and thus reduce overall code size.

The ‘Ex10 Core’ is a collection of modules required for communication with the Impinj reader chip, which includes modules from ‘Ex10Ops’ and ‘Ex10Protocol’.

The ‘Ex10 Support’ is a collection of modules includes a ramp module manager that is leveraged by the ‘Ex10RFPower’ reader primitive. Several support modules are included to provide functionality such as antenna disconnect and Listen before talk (LBT). There is also an example combining both antenna disconnect and LBT modules, which is referred to as a ‘super module’ example.

Other support modules are included to perform various diverse tasks such as command line parsing, aggregate operations and a printf wrapper.

The interface exposed by the ‘Use Case’ layer is called the ‘Impinj reader chip host API’.

The interface exposed by the Impinj reader chip is called the ‘Impinj reader chip wireline’.
