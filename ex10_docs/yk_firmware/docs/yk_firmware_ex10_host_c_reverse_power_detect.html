Reverse Power Threshold Detection¶

The return loss detection mechanism in the reader layer allows a way to check the current return loss and compare against a threshold.

This check can be called through the reader function get_return_loss_threshold_exceeded. This allows the user to utilize this functionality at any time they desire. This can be helpful in detecting if the current antenna has been disconnected, if the antenna is damaged, or if something unexpected has happened in the TX/RX pathway. Note that this function can only be called when an op is not running, so be sure to wait for completion of previous ops before calling.

A sequence diagram can be seen for the reverse power threshold detection call below…

In addition to calling this check through the reader layer when desired, there is a call to check the reverse power as a callback after each rampup. If the user uses the reader layer for ramp up, the integrated ramp-up process uses a reader layer callback function which get registered to the post_rampup_callback callback in the ops layer. The user can alter this callback to change the parameters/threshold to this check based on customer board specifications. If desired, this check can be removed entirely from the process.
