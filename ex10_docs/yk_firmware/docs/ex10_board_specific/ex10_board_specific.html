Board Specific ImplementationÂ¶

The board specific interfaces and constants, listed below, need to be changed based on CPU selection and connections to the Impinj reader chip. Board specific implementations are organized in the ex10_c_dev_kit/board directory. To add support for a new board, create a new board directory within ex10_c_dev_kit/board and implement the interface defined in the header files in the ex10_c_dev_kit/board directory. The Impinj E710 Development Board can be used as a reference and is found in the ex10_c_dev_kit/board/e710_ref_design directory.

This board specific layer is attempting to define a set of resources that the SDK defines that are used to interact with the Impinj reader chip, or control other aspects of the reader. A good example of this is the GPIO interface described below, as it has a setter for an antenna but it only takes an 8-bit number, and assumes that the board implementation knows how to decode that value to set the correct antenna. This could include setting one or more of the GPIO pins on the Impinj reader chip to a certain state, or could communicate with an external antenna mux through some other means, or some combination of both. To accommodate this flexibility it accepts the 8-bit number and then returns the GPIO pins to set (if any). The intention is to give the flexibility to the person implementing the reader around the Impinj reader chip, and not have the SDK make any assumptions about the environment, such as the number of antennas.
